# 我与世界久别重逢

> 別自残一生，別望前度在意。 <br/>
——黄伟文

## 西康路到安吉路的距离

我的建筑生涯暂时告一段落。2017年9月，我毫不知觉的开始了我人生最混乱的时期，当然，这只是当前（2024年2月）看来。显而易见，建筑设计尽管充满可能性，但莫名的不属于我想象中所喜欢的创造性工作的范畴。可怜我涉猎太窄，并不知道即便最纯粹的设计工作也充满着一种非创造性的束缚，何况建筑两个字给设计的压力绝不只是体量和规范的压力。因此，一种厌恶感在面对课设时激增，笼罩地整个生活天昏地暗。五年学生生涯艰难爬坡，为了减轻负重，放弃了很多原本拥有，对不起很多人。然而，除了适应了折磨，假装爱上建筑之外，没有一点收获，这实在令我后怕。入职后，我虚伪的“爱上”被工作彻底打回原形，我猛地发现，我已经一无所有了，又何必将仅剩的时间财富浪费在一个极度消磨我精神的事情上。而所谓离开的沉没成本，不过是多年折磨的白受罢了，其实我并没有获取到什么。

于是我毫不犹豫离开了，当骑行在南方乡下的田间野路，还有种挣脱牢笼一般自我感动。幸运的是，我有一双父母无条件愿意相信并支持我的选择。他们并没能亲历我的遭遇，但却能体谅我的痛苦。还有那些在歇斯底里的岁月里能陪我虚度的好友同窗。也许我想太多，矫作了，但还是想说如何朴素的情感表达，都无法描述我的感激。

那就不描述了。几乎一年时间的思考和梳理自身，该找到方向重新出发了。青春状态被消磨殆尽后，前路的风霜便不仅仅是风霜。我将这风霜隽文，写给谁谁。

## 这是我怀里抱着的书

建筑学生涯的学习习惯很差，也许是和建筑学本身的特点有关。没有笔记，没有系统的框架和了解。所有的知识都是走马观花，道听途说来的。什么都知道些，什么都不知道。自己的斤两只有自己知道，内行人面前当哑巴，外行人面前标榜艺术。很久之前便想改变了，但在建筑或者说设计上，一直没有找到一种合适的学习方法。

如今不再和PhotoShop，SketchUp，RHINO相伴，便又是一个塑造个人知识世界的机会。在试用和选择了若干方案（包括notion，obsidian，思源，flowus，语雀，Gatsby，前后端自己搭建等）之后，最终选择了Docusaurus和Github Pages结合的方案。搭配gh-pages的`action`方案，可以实现`git push`之后自动化部署。其优点结合了笔记应用的自动发布和自己搭建前后端的高自由度。当然，不如笔记应用的便捷性也不如自己搭建的自由度高。由此可见，妥协处处存在（在学过算法之后更觉如此），所以神仙鱼归天要怪谁？这也许也是我看好和选择rust的原因。

:::info 几种知识库或博客的比较
个人对市面上知识库的一些经验：各有优缺点，我以前也固执的寻找一款近乎完美的软件，但很遗憾。后来发现，像是量子力学中的不确定性原理一般，粒子的位置与动量不可同时被确定；时间复杂度和空间复杂度不能同时优先；篮球运动员的身高和灵活性不能同时很强（有人会说文班亚马，但他的灵活性只是相对于他的身高来说比较强）。这堆软件的所有的优点也不可能同时满足。
1. 我使用最多的其实是语雀。界面好看，云端同步，块类型很多，感觉很舒服。缺点是没有本地化，速度就会慢，尤其是文档内容很大之后，或是公式很多，非常卡。其次移动端没有编辑能力，功能损失严重。
2. 第二多的是flowus。界面也比较好看，块类型也很多，不卡。缺点是同步空间太小，会员是针对空间扩容。
3. 思源是一个开源软件。跟obsidian比起来更容易上手。非常好用。社区也很好，更新也频繁。
4. notion由于网络问题，在试用几天后便弃用了。从网络来讲，体验很差。而且个人观感上不如flowus好看。
5. Gatsby也不错，自建博客框架，容易上手。但是GraphQL我觉得不好用。
6. 我之前还用过很多自建的后端，包括嵌入式数据库和MySql，但是维护成本很大，很多精力都会浪费在维护上，久而久之便放弃了。
:::

## 坎特维尔的幽灵

该栏目下，记录的主要是计算机相关的内容。
1. 算法和数据结构：包括基本原理和LeetCode实践两大部分。
2. 计算机组成原理：主要是CPU和内存原理，编译原理等。
3. 操作系统基础：进程和线程在异步环境下的作用和工作方法，文件系统和设备控制等。
4. 数据库原理：几种主流数据库的实际应用。
5. 计算机网络：TCP/IP协议的学习，应用层各类协议的学习。
6. 必要的数学知识：微积分、线性代数和数理统计。
7. 语言学习：主要是rust和python的学习。
