致明日的舞

別自残一生，別望前度在意。
——黄伟文

躺于瘠土一片​

我的建筑生涯暂时告一段落。并不是说我跟建筑老死不相往来，但就跟谈恋爱一样，在没有彻底放下之前，经不住再次见面。

2017年9月，我毫不知觉的开始了我人生最兵荒马乱的时期，当然，这只是当下（2024年2月）的阶段性总结。现在谈来，我甚至不能准确的解释究竟什么令我不爽甚至痛苦。尽管我在外人面前惴惴掩饰慌乱，但拙劣的演技并不能带来事实的从容。后来我经常试图通过分析自己憎恶的原因来慰籍平复心情和用口头承认自己的不堪来暗示自己放得下，但结果不得而知。

建筑设计尽管充满可能性，但莫名的不属于我想象中所喜欢的创造性工作的范畴。可怜我涉猎太窄，并不知道即便最纯粹的设计工作也充满着一种非创造性的束缚（我理想中的自由虽然没有否认规则和环境的约束，但仍然被这种事实上的非创造性束缚恶心到。显而易见对我而言这两种的束缚并不相同，我有时会归结为它的合理性，有时却又否认），何况建筑两个字给设计的压力绝不只是体量和规范的压力（这中压力可能跟接近于我认可的规则，而设计本身和审美上的潜规则似乎才是最终击败我的那位）。因此，一种厌恶感在面对课设时激增，笼罩地整个生活天昏地暗。五年学生生涯艰难爬坡，为了减轻负重，放弃了很多原本拥有，对不起很多人。然而，除了适应了折磨，假装爱上建筑之外，没有一点收获，这实在令我后怕。入职后，我虚伪的“爱上”被工作彻底打回原形，我猛地发现，我已经一无所有了，又何必将仅剩的时间财富浪费在一个极度消磨我精神的事情上。而所谓离开的沉没成本，不过是多年折磨的白受罢了，其实我并没有获取到什么。

于是我毫不犹豫离开了，当骑行在江南乡下的田间野路，还有种挣脱牢笼一般自我感动。幸运的是，我有一双父母无条件愿意相信并支持我的选择。他们并没能亲历我的遭遇，但却能体谅我的痛苦。

我试图通过考研（2024）去跨行业，但成绩不理想，数学尽管多年未碰，但自觉复习效果还行。直到卷子到手，略读就觉大事不妙。难度重点考法都与以往不同，计算量也大。果然考后哀鸿遍网，都说难度长年罕见。

也许是已经经历了很多了，我也不觉失落，只感觉委屈。抱怨又怕别人说是借口。这一年练了很多题，能做的基本都做了，结果记的、背的、练的都没用上。我自知荒废多年，底子已经很差了，目标分数下限定的也不高。可如今离下限都有很大的距离。发现命运早就定了，上帝早就知道盒子里的猫是死是活。

既如此，接下来只可以更加激进。

雨再降，也觉得不冻​

建筑学生涯的学习习惯很差，也许是和建筑学本身的特点有关。没有笔记，没有系统的框架和了解。所有的知识都是走马观花，道听途说来的。什么都知道些，什么都不知道。自己的斤两只有自己知道，内行人面前当哑巴，外行人面前标榜艺术。很久之前便想改变了，但在建筑或者说设计上，一直没有找到一种合适的学习方法。

如今不再和PhotoShop，SketchUp，RHINO相伴，便又是一个塑造个人知识世界的机会。在试用和选择了若干方案（包括notion，obsidian，思源，flowus，语雀，Gatsby，前后端自己搭建等）之后，最终选择了Docusaurus和Github Pages结合的方案。搭配gh-pages的action方案，可以实现git push之后自动化部署。其优点结合了笔记应用的自动发布和自己搭建前后端的高自由度。当然，不如笔记应用的便捷性也不如自己搭建的自由度高。由此可见，妥协处处存在（在学过算法之后更觉如此），所以神仙鱼归天要怪谁？这也许也是我看好和选择rust的原因。

几种知识库或博客的比较

个人对市面上知识库的一些经验：各有优缺点，我以前也固执的寻找一款近乎完美的软件，但很遗憾。后来发现，像是量子力学中的不确定性原理一般，粒子的位置与动量不可同时被确定；时间复杂度和空间复杂度不能同时优先；篮球运动员的身高和灵活性不能同时很强（有人会说文班亚马，但他的灵活性只是相对于他的身高来说比较强）。这堆软件的所有的优点也不可能同时满足。

  1. 我使用最多的其实是语雀。界面好看，云端同步，块类型很多，感觉很舒服。缺点是没有本地化，速度就会慢，尤其是文档内容很大之后，或是公式很多，非常卡。其次移动端没有编辑能力，功能损失严重。
  2. 第二多的是flowus。界面也比较好看，块类型也很多，不卡。缺点是同步空间太小，会员是针对空间扩容。
  3. 思源是一个开源软件。跟obsidian比起来更容易上手。非常好用。社区也很好，更新也频繁。
  4. notion由于网络问题，在试用几天后便弃用了。从网络来讲，体验很差。而且个人观感上不如flowus好看。
  5. Gatsby也不错，自建博客框架，容易上手。但是GraphQL我觉得不好用。
  6. 我之前还用过很多自建的后端，包括嵌入式数据库和MySql，但是维护成本很大，很多精力都会浪费在维护上，久而久之便放弃了。

该栏目下，记录的主要是计算机相关的内容。

  1. 算法和数据结构：包括基本原理和LeetCode实践两大部分。
  2. 计算机组成原理：主要是CPU和内存原理，编译原理等。
  3. 操作系统基础：进程和线程在异步环境下的作用和工作方法，文件系统和设备控制等。
  4. 数据库原理：几种主流数据库的实际应用。
  5. 计算机网络：TCP/IP协议的学习，应用层各类协议的学习。
  6. 必要的数学知识：微积分、线性代数和数理统计。
  7. 语言学习：主要是c++,rust和python的学习。

这个你已叫天清朗​

功败也极乐，我亦坐佛莲。功能主义地一边学习，问题导向地边找项目，由小到大，由简到难，吸取教训，早日财富自由。如今几乎一年时间的思考和梳理自身（事实上这些年我无时无刻不在做这件事，但每每被糟事打断，也便没有下文），该找到方向重新出发了。青春状态被消磨殆尽后，前路的风霜便不仅仅是风霜。我将这风霜隽文，写给谁谁。
